## 基本

このマニュアルページは GNU 版の `chown` について説明しています。
`chown` は、指定された各ファイルのユーザーおよび/またはグループの所有者を変更する。
所有者（ユーザー名または数値のユーザー ID）のみが指定された場合、そのユーザーが指定された各ファイルの所有者となり、ファイルのグループは変更されない。
所有者の後にコロンとグループ名（または数値のグループ ID）を空白なしで続けた場合、ファイルのグループ所有権も変更される。

ユーザー名の後にコロンが続き、グループ名がない場合、そのユーザーがファイルの所有者となり、ファイルのグループはそのユーザーのログイン・グループに変更される。
コロンとグループが与えられ、所有者が省略された場合、ファイルのグループのみが変更される。
コロンのみが指定された場合、またはオペランド全体が空の場合、所有者もグループも変更されない。

### 構文

```bash frame=none
chown [OPTION]... [OWNER][:[GROUP]] FILE...
chown [OPTION]... --reference=RFILE FILE...
```

- **OWNER**: 新しい所有者のユーザー名または UID。
- **GROUP**: 新しいグループのグループ名または GID。
- **FILE**: 所有者またはグループを変更するファイルまたはディレクトリ。

### EXAMPLES

```bash frame=none
# Change the owner of /u to "root".
chown root /u

# Likewise, but also change its group to "staff".
chown root:staff /u

# Change the owner of /u and subfiles to "root".
chown -hR root /u
```

`OWNER` と `GROUP` は、記号だけでなく数値でもよい。

複数指定した場合は、最後の 1 つだけが有効になる。

### SAMPLE

#### 単一のファイル・ディレクトリの所有者・グループを変更する

```bash frame=none
# 単一のファイルの所有者を変更する
chown newowner filename

# 単一のディレクトリの所有者を変更する
chown newowner directoryname

# 単一のファイルの所有者とグループを変更
chown newowner:newgroup filename

# 単一のディレクトリの所有者とグループを変更
chown newowner:newgroup directoryname

# 所有者を変更せずに、単一のファイルのグループのみを変更
chown :newgroup filename
```

`chown` コマンドは、通常、スーパーユーザー（root）として実行する必要があります。また、`chown` コマンドには他にも多くのオプションがあります。
詳細については、`man chown` コマンドを使用してマニュアルページを参照するか、オンラインのマニュアルをチェックしてください。

#### `-R` オプション

##### ディレクトリとその内容の所有者とグループを再帰的に変更する

```bash frame=none
chown -R newowner:newgroup directoryname
sudo chown -R ec2-user:ec2-user .git
```

##### 所有者を変更せずに、ディレクトリとその内容のグループを再帰的に変更する

```bash frame=none
chown -R :newgroup directoryname
```
